# Week 1 – Understanding the Business Problem

## Project Title
**E-Commerce Order Management Database System**

## Objective
The objective of Week 1 is to analyze the business requirements of an E-Commerce Order Management Database System and prepare a Software Requirement Specification (SRS). This phase focuses on understanding the business scenario, identifying stakeholders, defining system requirements, and documenting the project scope.

## Documents Included

- Requirement_Analysis_Report.pdf
- Business_Requirement_Document.pdf
- SRS_Document.pdf

## Topics Covered

- Business Scenario Analysis
- Core Entity Identification
- Stakeholder Analysis
- Functional Requirements
- Non-Functional Requirements
- Project Scope
- Assumptions and Constraints
- Software Requirement Specification (SRS)

## Core Entities

- Customer
- Category
- Product
- Supplier
- Order
- Order Details
- Payment
- Shipment
- Review

## Learning Outcomes

- Understand business requirements.
- Analyze an E-Commerce business process.
- Prepare professional software documentation.
- Identify system requirements before database design.
