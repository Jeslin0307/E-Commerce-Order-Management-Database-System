# Week 2 – Entity and Relationship Analysis

## Project Title
## E-Commerce Order Management Database System

## Objective
The objective of Week 2 is to analyze the data requirements of the E-Commerce Order Management Database System by identifying entities, attributes, primary keys, foreign keys, relationships, and cardinality. This analysis serves as the foundation for Entity Relationship Diagram (ERD) design and database implementation.

## Topics Covered

- Entity Identification
- Entity Attribute Analysis
- Primary Key Identification
- Foreign Key Identification
- Database Constraints
- Relationship Analysis
- Cardinality Analysis
- Business Rules

- Customer
- Category
- Product
- Supplier

- Order Details
- Payment
- Shipment
- Review

## Learning Outcomes

- Understand database entities.
- Identify primary and foreign keys.
- Analyze relationships between entities.
- Determine relationship cardinality.
- Prepare database design documentation.
